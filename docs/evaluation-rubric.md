# 기사 게재 평가 루브릭

이 문서는 자동 생성 기사가 사이트에 게재(`status: "published"`)되기 전 통과해야 하는
평가 기준을 정의한다. `scripts/generate-news.sh`가 이 문서를 읽어 `claude -p`에
프롬프트로 주입한다.

---

## 패스 게이트 규칙

기사가 `status: "published"`가 되려면 **두 조건을 모두** 만족해야 한다.

1. **하드 게이트 6개 모두 통과** (아래 H1~H6)
2. **사실 정확성(15점)을 제외한 모든 루브릭 항목이 만점**
   - 즉, 사실 정확성에서만 감점이 허용되고, 다른 항목에서 단 0.5점이라도 감점되면 실패
   - 통과 시 총점 범위: 85~100 (사실 정확성 0~15 + 비사실 항목 만점 85)

게이트에 실패한 기사는 `status: "draft"`로 저장하며, `evaluation.gate = "fail"`,
`evaluation.notes`에 감점 사유를 기록한다.

---

## 하드 게이트 (자동 탈락)

| # | 항목 | 탈락 기준 |
|---|------|----------|
| H1 | 허위·조작 사실 | 검증 불가능한 주장 1건 이상 |
| H2 | 원문 복사 | 소스에서 20단어 이상 그대로 옮김 |
| H3 | 이중언어 누락 | EN 또는 KO body가 비어 있거나 누락 |
| H4 | 단어 수 극단 이탈 | 각 언어 200단어 미만 또는 800단어 초과 |
| H5 | JSON 스키마 위반 | 필수 필드 누락, 타입 오류, JSON 파싱 실패 |
| H6 | enum 위반 | `category`가 정의되지 않은 값, 태그가 0개 또는 9개 이상 |

## 점수 루브릭 (총 100점)

### 1. 콘텐츠 무결성 (30점)
| 항목 | 배점 | 만점 기준 |
|------|------|----------|
| 사실 정확성 | 15 | 모든 사실 주장이 sourceUrls에서 검증 가능 (이 항목만 감점 허용) |
| 원본성 | 10 | 원문 표현·구조를 자기 언어로 완전히 재구성 |
| 출처 표기 | 5 | sourceUrls 배열에 정확한 원본/참고 URL 1개 이상 (HTTP 200 응답) |

### 2. 언어 품질 (25점)
| 항목 | 배점 | 만점 기준 |
|------|------|----------|
| 한국어 자연성 | 10 | 한국인 기자가 직접 쓴 듯한 문장, 번역체 없음 |
| 영어 저널리즘 톤 | 10 | 전문 기자 수준 어휘·문법·흐름 |
| 양 언어 일치성 | 5 | EN/KO가 같은 사실·결론을 전달 (직역 아님, 자연스러운 동치) |

### 3. 구조와 포맷 (15점)
| 항목 | 배점 | 만점 기준 |
|------|------|----------|
| 단어 수 | 5 | EN 300~500 단어, KO 300+ 어절 |
| HTML 포맷팅 | 5 | `<p>`, `<h3>`, `<strong>`, `<em>` 등 적절히 사용, 가독성 확보 |
| 카테고리·태그 적합성 | 5 | 카테고리 1개 정확 (enum 일치), 태그 3~6개 모두 관련성 있음 |

### 4. SEO·메타데이터 (10점)
| 항목 | 배점 | 만점 기준 |
|------|------|----------|
| 제목 | 3 | EN 60자 이하, KO 40자 이하, 구체적이며 낚시성 아님 |
| summary | 3 | 1~2 문장, 160자 이하, 본문 핵심 전달 |
| metaDescription | 2 | 155자 이하, 키워드 포함 |
| slug | 2 | URL 친화적 (소문자·하이픈), 핵심 키워드 반영 |

### 5. 도메인 가치 (10점)
| 항목 | 배점 | 만점 기준 |
|------|------|----------|
| 주제 적합성 | 5 | 다이빙/프리다이빙 범위에서 벗어나지 않음 |
| 독자 효용 | 5 | 다이버가 읽고 얻는 인사이트·실천 팁·뉴스 가치 명확 |

### 6. 안전·윤리 (10점)
| 항목 | 배점 | 만점 기준 |
|------|------|----------|
| 안전 콘텐츠 면책 | 3 | 의료·법률 조언 어조 회피, 공인 강사·교육 기관 권장 명시 |
| 사고·사망 보도 톤 | 3 | 센세이셔널 아님, 피해자·가족 존중, 사실만 보도 |
| 이미지 관련성 | 2 | imageUrl이 주제와 어긋나지 않음, 다른 기사와 중복 없음 |
| 추측 vs 사실 구분 | 2 | "~로 보인다", "~일 가능성" 등 헷지 어휘 사용 |

## 평가 결과 저장 형식

기사 JSON에 `evaluation` 필드를 추가한다. TypeScript 타입은
[src/lib/types.ts](../src/lib/types.ts)의 `ArticleEvaluation` 참조.

```json
{
  "evaluation": {
    "score": 100,
    "gate": "pass",
    "evaluatedAt": "2026-04-19T12:10:00Z",
    "notes": "Round 2에서 통과. Round 1에서 출처 표기·제목 길이 감점을 수정.",
    "breakdown": {
      "contentIntegrity": 30,
      "languageQuality": 25,
      "structureFormat": 15,
      "seoMetadata": 10,
      "domainValue": 10,
      "safetyEthics": 10
    }
  }
}
```

## 자동 파이프라인의 평가 루프

`scripts/generate-news.sh`는 각 신규 기사에 대해 다음을 수행한다.

```
초안 작성 (status="draft")
  ↓
하드 게이트 6개 체크
  ├─ 실패: status="draft" 유지, evaluation 기록, 다음 기사로
  └─ 통과 ↓
점수 채점 (100점)
  ↓
게이트 판정 (사실 정확성 외 모두 만점인가?)
  ├─ 통과: status="published", evaluation 기록, 완료
  └─ 실패 ↓
감점 항목 분석 → 기사 수정 → 재채점
  ↓
최대 3 라운드까지 반복
  ↓
3 라운드 후에도 실패: status="draft" 유지, 마지막 evaluation 기록 (수동 검토 대기)
```
